Build a mobile-first web app called Fortify — a supplemental strength training app for functional athletes (CrossFit, Hyrox, and ATHX competitors). The app is a PWA (progressive web app) that should feel native on iPhone and Android.
Stack: React + Vite + Tailwind CSS + wouter for routing. Do not use any Replit-specific services, connectors, databases, auth providers, or secrets managers. Do not use localStorage for persistent data. All persistent data lives in a standalone Supabase project connected via environment variables.

ENVIRONMENT VARIABLES
Read exclusively from import.meta.env. Never hardcode credentials.
Create a .env.example file:
VITE_SUPABASE_URL=your-supabase-project-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
Connect using @supabase/supabase-js. The Supabase client handles session token persistence in localStorage automatically — do not manually manage auth tokens.

DATABASE SCHEMA
Generate supabase/migrations/001_initial.sql with the following tables, RLS policies, and indexes:
sql-- Profiles (extends Supabase auth.users)
create table profiles (
  id uuid references auth.users primary key,
  name text not null,
  sport text not null,
  subtrack text not null,
  level text not null,
  frequency int not null,
  is_beta boolean default true,
  created_at timestamptz default now()
);
alter table profiles enable row level security;
create policy "profiles_own" on profiles for all using (auth.uid() = id);

-- Workout sessions
create table sessions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id) on delete cascade,
  sport text not null,
  subtrack text not null,
  week_number int not null,
  day_number int not null,
  workout_id text not null,
  notes text,
  completed_at timestamptz default now()
);
alter table sessions enable row level security;
create policy "sessions_own" on sessions for all using (auth.uid() = user_id);

-- Set logs
create table set_logs (
  id uuid primary key default gen_random_uuid(),
  session_id uuid references sessions(id) on delete cascade,
  exercise_name text not null,
  movement_id text,
  set_number int not null,
  weight_lbs numeric,
  reps int,
  rpe_actual numeric,
  logged_at timestamptz default now()
);
alter table set_logs enable row level security;
create policy "set_logs_own" on set_logs for all using (
  auth.uid() = (select user_id from sessions where id = session_id)
);

-- Personal records
create table personal_records (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id) on delete cascade,
  movement text not null,
  weight_lbs numeric not null,
  notes text,
  achieved_at date not null,
  created_at timestamptz default now()
);
alter table personal_records enable row level security;
create policy "prs_own" on personal_records for all using (auth.uid() = user_id);

-- Movement library
create table movements (
  id text primary key,
  name text not null,
  category text not null,
  subcategory text,
  tags text[] default '{}',
  youtube_url text,
  youtube_embed_id text,
  cue_points text[],
  description text,
  difficulty text,
  equipment text[],
  primary_muscles text[],
  secondary_muscles text[],
  is_active boolean default true,
  created_at timestamptz default now()
);
alter table movements enable row level security;
create policy "movements_public_read" on movements for select using (true);

-- Workout templates
create table workouts (
  id text primary key,
  sport text not null,
  subtrack text not null,
  week_number int not null,
  day_number int not null,
  title text not null,
  coach_note text,
  warmup jsonb,
  main_work jsonb,
  accessory jsonb,
  created_at timestamptz default now()
);
alter table workouts enable row level security;
create policy "workouts_public_read" on workouts for select using (true);

-- Community chat
create table chat_messages (
  id uuid primary key default gen_random_uuid(),
  subtrack text not null,
  user_id uuid references profiles(id) on delete cascade,
  author_name text not null,
  content text not null,
  is_coach boolean default false,
  is_pinned boolean default false,
  created_at timestamptz default now()
);
alter table chat_messages enable row level security;
create policy "chat_read" on chat_messages for select using (true);
create policy "chat_insert_own" on chat_messages for insert with check (auth.uid() = user_id);

-- Feed likes
create table feed_likes (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id) on delete cascade,
  feed_item_id text not null,
  created_at timestamptz default now(),
  unique(user_id, feed_item_id)
);
alter table feed_likes enable row level security;
create policy "likes_read" on feed_likes for select using (true);
create policy "likes_own" on feed_likes for all using (auth.uid() = user_id);
Enable Supabase Realtime on chat_messages and sessions tables.

AUTH FLOW
Use Supabase Auth (email + password). On first launch with no active session show two options: Create Account and Sign In.
Create Account runs the full onboarding flow: sport selection → sub-track selection → experience level → training frequency → name, email, and password. On completion call supabase.auth.signUp() then insert the profile row.
Sign In shows email and password fields and calls supabase.auth.signInWithPassword().
Add Sign Out in the Profile tab calling supabase.auth.signOut().

APP STRUCTURE — 5 bottom nav tabs

Home (today's workout)
Program (calendar + cycle overview)
PRs (personal record tracker)
Community (feed + chat)
Profile (includes movement library link)


HOME TAB
On load, calculate the user's current week and day based on profiles.created_at and their frequency setting. Query workouts for the matching sport + subtrack + week_number + day_number. Show skeleton loaders while fetching.
Show a header card: sport, subtrack name, week number, day number.
Render the workout from Supabase:

coach_note field as a highlighted card above the workout
Warmup section from warmup JSONB (static list, no logging)
Main Work from main_work JSONB
Accessory from accessory JSONB

For each exercise in Main Work and Accessory:

Name, sets × reps, RPE target
Expandable log panel: one row per set with weight (lbs), reps, and optional RPE inputs
A "?" icon that opens a modal showing the exercise description, cue points from the linked movement, primary and secondary muscles as tags, and a YouTube embed using https://www.youtube.com/embed/[youtube_embed_id] from the movements table (look up by movement_id)
Rest timer button using rest_seconds from the JSONB — shows a countdown bottom sheet when active

Finish Workout button:

Inserts row into sessions
Inserts all logged sets into set_logs
Checks if any logged weight exceeds existing personal_records for that movement — if so shows a "New PR!" prompt
Shows a completion screen

Show a rest day card if no workout matches today.

PROGRAM TAB
Horizontal week strip (Mon–Sun). Scheduled days highlighted. Completed days (from sessions query) show a checkmark.
List all weeks of the current cycle as expandable rows. Tap a past day to load that workout in read-only mode showing what was logged.
Show cycle metadata: sport, subtrack, total weeks, current week, sessions completed vs planned.

PRs TAB
Query personal_records for the current user. Searchable list: movement name, best weight, date.
Tap any movement to see a recharts line chart of all logged entries over time.
"+ Add PR" button: movement type-ahead from the movements table (query by name), weight, date, notes. Inserts into personal_records.
Movement type-ahead must query the movements table — not a hardcoded array.

COMMUNITY TAB
Two sub-tabs: Feed and Chat.
Feed: Query the 50 most recent sessions joined with profiles. Also query recent personal_records. Merge and sort by timestamp. Each card shows athlete name, action, sport/subtrack tag, relative time, and a like button that inserts/deletes from feed_likes.
Chat: Tab bar of sub-tracks for the user's sport. Default to the user's current subtrack. Query chat_messages filtered by subtrack. Subscribe via Supabase Realtime so new messages appear without refresh. Pinned messages (is_pinned = true) render at the top. Coach messages (is_coach = true) show a "Coach" badge. Text input and send button inserts into chat_messages with the current user's id and name.

MOVEMENT LIBRARY
Accessible from the Profile tab as a full screen. Contains:

Search bar querying movements.name with case-insensitive match
Filter chips for category: squat, hinge, press_vertical, press_horizontal, pull_vertical, pull_horizontal, carry, olympic, explosive, core, accessory
Filter chips for sport tag: crossfit, hyrox, athx
Scrollable list of matching movements showing name, category badge, difficulty badge, primary muscles
Tapping a movement opens a detail view: description, cue points, equipment, primary and secondary muscles, and a YouTube embed using youtube_embed_id

All movement queries run against the movements Supabase table. No hardcoded movement arrays anywhere in the app.

PROFILE TAB
Query profiles for the current user. Show name, sport/subtrack badge, Beta Member badge, joined date, sessions completed (count from sessions), PRs logged (count from personal_records).
Settings: switch track option with a warning modal ("Switching tracks resets your current cycle. Your PR history is kept.") that updates the profiles row. Sign Out button. Link to Movement Library.
"Powered by 2POOD" footer.

DESIGN

Mobile-first. Max width 430px centered on desktop, full-width on mobile
Dark background #0f0f0f. Card surfaces #1a1a1a and #1e1e1e
Accent color #F05A28 (bold orange) for CTAs, active nav tab, PR badges, sport tags
Bold athletic typography. Workout titles 24–28px. Body text 14–15px
Bottom nav with icons and labels. Active tab uses accent color
Skeleton loaders while Supabase queries are in flight — no spinners
Smooth tab transitions. Smooth expand/collapse on set logging panels
Rest timer as a prominent bottom sheet when active
The app should feel like something an athlete would want to open every day
